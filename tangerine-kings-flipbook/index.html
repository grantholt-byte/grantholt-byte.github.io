Tangerine Kings flipbook

Render the music video for "The Failing" on your GPU rig. Free.

What this is

You're rendering an animated music video for my fake 90s emo band, Tangerine Kings. Hand-drawn stick figures on cream paper. ~3:18 long. Output: an MP4 you send back to me.

What you'll need

  • Linux or WSL with at least one NVIDIA GPU (12 GB+ VRAM minimum, 24 GB+ recommended)
  • ~80 GB free disk
  • Hugging Face account (for Flux.1 [dev] license — free, one-time)
  • Python 3.11+, git, ffmpeg

One block. Paste. Walk away.

git clone https://github.com/grantholt-byte/grantholt-byte.github.io.git
cd grantholt-byte.github.io/tangerine-kings-flipbook
export HF_TOKEN=hf_...
bash run.sh

Get an HF_TOKEN: visit FLUX.1-dev on Hugging Face, accept the license, then create a token at settings/tokens.

If anything crashes mid-run, just re-run bash run.sh. It resumes from where it left off — completed stages are skipped.

What it'll do

[setup]      install ComfyUI + Flux + LoRA + RIFE      ~10–30 min one-time
[detect]     auto-tune for your specific GPU(s)        ~5 sec
[analyze]    librosa + WhisperX on the song            ~30 sec (first run + 3 GB download)
[schedule]   build per-keyframe shot schedule          ~2 sec
[render]     Flux+LoRA keyframes (multi-GPU)           ~25–60 min
[interpolate] RIFE 3× (8 fps → 24 fps)                  ~10 min
[assemble]   ffmpeg + paper post-processing            ~2 min

If something breaks

  • HF_TOKEN not set — see paragraph above the big block.
  • OOM on a card — set CUDA_VISIBLE_DEVICES=0 (or whichever cards have headroom) and re-run.
  • ComfyUI port collision (8188 in use) — kill the existing instance, then re-run.
  • Render output looks weird — open output/timing_map.json, fix section boundaries by hand, then rm output/.stage_done/render output/.stage_done/schedule and re-run. Only changed frames re-render.
  • Anything else — text me the last 30 lines of the terminal output.

Or run stages manually

Same things run.sh does, just one at a time:

bash setup.sh                                        # one-time
python scripts/0_detect.py                           # preflight + tune
python scripts/1_analyze_audio.py audio/TheFailing.mp3
python scripts/2_build_schedule.py
python scripts/3_render_keyframes.py
python scripts/4_interpolate_rife.py
python scripts/5_assemble_video.py

Output

Final file lands at output/the_failing_flipbook.mp4. Send it however you want — Dropbox, Telegram, AirDrop, USB stick, attached to a postcard.
